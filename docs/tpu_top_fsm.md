# Top-Level FSM (tpu_top) — Implementation Plan

`tpu_top.sv` is the synthesis top-level entity that instantiates every datapath module, drives the sequencer FSM, and connects to the DE1-SoC physical pins.

---

## File

`rtl/tpu_top.sv`

---

## Module Hierarchy

```
tpu_top.sv
├── host_interface.sv        (LWH2F AXI4-Lite CSR slave)
├── weight_loader.sv         (ROM → weight_fifo)
│   └── weight_rom           (M10K, initialized via $readmemh)
├── weight_fifo.sv           (double-buffered weight tile store)
│   └── fifo.sv (×4)
├── systolic_data_setup.sv   (skews activation rows into MMU)
├── mmu.sv                   (2×2 systolic array)
│   └── pe.sv (×4)
├── accumulator.sv           (de-skews MMU partial sums)
│   └── fifo.sv (×2)
├── bias.sv                  (adds per-column bias)
├── activation.sv            (ReLU)
├── unified_buffer.sv        (double-banked on-chip activation store)
└── seg7_decoder.sv          (4-bit digit → 7-segment encoding)
```

## Top-Level Port Interface (DE1-SoC Physical Pins)

```systemverilog
module tpu_top (
    // --- Clocks ---
    input  logic        CLOCK_50,          // 50 MHz main clock (PIN_AF14)

    // --- Reset / control buttons ---
    input  logic [3:0]  KEY,               // active-low; KEY[0]=reset, KEY[1]=manual start

    // --- Status LEDs ---
    output logic [9:0]  LEDR,              // active-high

    // --- 7-segment displays (active-low segments) ---
    output logic [6:0]  HEX0,             // predicted digit ones place
    output logic [6:0]  HEX1,             // predicted digit tens place (unused for 0-9)
    output logic [6:0]  HEX2,
    output logic [6:0]  HEX3,
    output logic [6:0]  HEX4,
    output logic [6:0]  HEX5,

    // --- Slide switches (debug / mode select) ---
    input  logic [9:0]  SW,

    // --- Lightweight HPS-to-FPGA AXI4-Lite bridge ---
    // (These ports come from Platform Designer HPS component)
    input  logic        hps_fpga_axi_clk,
    // ... AXI4-Lite slave signals wired from the Qsys-generated interconnect
);
```

**Note:** The exact AXI4-Lite port list is generated by Quartus Platform Designer when the HPS component is instantiated. The `tpu_top` top-level imports the Qsys-generated `.qip` and connects the `host_interface` module as the AXI slave.

## Sequencer FSM

The FSM orchestrates a single inference pass (one input matrix through one or more layers). It lives inside `tpu_top` rather than in a separate module because it directly controls signals that span multiple instantiated modules.

### States

```
IDLE
  Wait for host START or KEY[1] press.
  LEDR = 10'b0000_0000_01 (idle indicator).

LOAD_WEIGHTS
  Assert weight_loader.start_load with tile_base_addr for current (layer, tile_k) tile.
  Wait for weight_loader.done.
  Then: pulse weight_fifo.swap_banks for 1 cycle (shadow → active).
  Next → WEIGHT_DRAIN.

WEIGHT_DRAIN
  Assert loading_phase = 1.
  Hold until weight_fifo.active_empty = 1 (all weights drained into MMU PEs).
  Deassert loading_phase.
  Next → STREAM_ACT.

STREAM_ACT
  For each row r in 0..ARRAY_ROWS-1:
    Assert unified_buffer.ub_read_en with ub_read_addr = r.
    Wait 1 cycle (UB latency stage 1).
    Wait 1 cycle (UB latency stage 2 — data on ub_read_data).
    ub_read_data flows into systolic_data_setup, which handles the row skew automatically.
  After all rows issued: next → WAIT_DRAIN.

WAIT_DRAIN
  Wait for the pipeline tail to flush:
    ARRAY_ROWS + (ARRAY_SIZE - 1) + 2 cycles
    (ARRAY_SIZE-1 for systolic skew, +2 for bias+activation pipeline stages)
  During this time, accumulator, bias, activation are all in flight.
  activation.out_row_valid pulses land in unified_buffer.act_write_*.
  Next → CHECK_TILE.

CHECK_TILE
  if tile_k < num_k_tiles - 1:
    Increment tile_k_idx.
    Next → LOAD_WEIGHTS  (next K-dimension tile; accumulator sums across passes)
  else:
    tile_k_idx = 0.
    Next → CHECK_LAYER.

CHECK_LAYER
  if layer_idx < num_layers - 1:
    Increment layer_idx.
    Pulse unified_buffer.bank_swap (output bank becomes new input bank).
    Reset unified_buffer.act_write_addr_reset.
    Next → LOAD_WEIGHTS  (first tile of next layer)
  else:
    Next → DONE.

DONE
  Assert STATUS.DONE (visible via host_interface STATUS register).
  Drive HEX0 with seg7_decoder(argmax(output_row)).
  LEDR = 10'b0000_0001_00 (done indicator).
  Wait for host CTRL.START deassert or CTRL.RESET before returning to IDLE.
```

### State Diagram

```
IDLE → LOAD_WEIGHTS → WEIGHT_DRAIN → STREAM_ACT → WAIT_DRAIN → CHECK_TILE
                                                                      │
                                           ←─── (more K tiles) ──────┘
                                                                      │
                                                               CHECK_LAYER
                                                                      │
                                           ←─── (more layers) ────────┘
                                                                      │
                                                                    DONE → IDLE
```

## Key Counters and Registers

| Register | Width | Description |
|---|---|---|
| `tile_k_idx` | `$clog2(MAX_K_TILES)` | Current horizontal tile through the K (input feature) dimension |
| `layer_idx` | `$clog2(MAX_LAYERS)` | Current network layer being computed |
| `drain_timer` | `$clog2(DRAIN_CYCLES+1)` | Countdown for WAIT_DRAIN state |
| `stream_row_idx` | `$clog2(ARRAY_ROWS)` | Row counter in STREAM_ACT state |

**Drain cycle calculation:**
```
DRAIN_CYCLES = ARRAY_ROWS           // rows to stream
             + (ARRAY_SIZE - 1)     // systolic skew latency
             + 1                    // accumulator FIFO pop latency
             + 1                    // bias registered latency
             + 1                    // activation registered latency
             + 2                    // UB write latency margin
```

For the 2×2 prototype: `DRAIN_CYCLES = 2 + 1 + 1 + 1 + 1 + 2 = 8` cycles.

## Tiling for Large Matrices

For MNIST 784→128 with an 8×8 array:
- K dimension: `ceil(784/8) = 98` tile passes per output tile
- Output tiles: `ceil(128/8) = 16` column tiles
- Total tile passes per layer: `98 × 16 = 1568`

During tiled passes, the accumulator's per-column FIFOs collect partial sums from each pass. The accumulator's `pop_row` logic naturally holds results until all column FIFOs are non-empty simultaneously — which happens after all K-dimension passes have contributed to each output column. **No change to accumulator.sv is needed** as long as:

1. The FSM does not drain the UB (write to output bank) until all K-dimension passes are complete for a given output tile.
2. The accumulator FIFO depth is ≥ number of output rows (currently `FIFO_DEPTH=4`, sufficient for a 2×2 or 4×4 array).

For larger arrays: increase `FIFO_DEPTH` parameter when instantiating the accumulator.

## 7-Segment Argmax

After the last layer completes, the output row in the UB contains 10 logit values (for MNIST). The FSM reads these via `ub_read_*`, computes the argmax in a small combinational circuit, and drives `seg7_decoder`:

```systemverilog
// rtl/seg7_decoder.sv — 4-bit digit to 7-segment (active-low)
module seg7_decoder (
    input  logic [3:0] digit,
    output logic [6:0] seg    // {g,f,e,d,c,b,a}, 0 = segment ON
);
    always_comb begin
        case (digit)
            4'd0: seg = 7'b100_0000;
            4'd1: seg = 7'b111_1001;
            4'd2: seg = 7'b010_0100;
            4'd3: seg = 7'b011_0000;
            4'd4: seg = 7'b001_1001;
            4'd5: seg = 7'b001_0010;
            4'd6: seg = 7'b000_0010;
            4'd7: seg = 7'b111_1000;
            4'd8: seg = 7'b000_0000;
            4'd9: seg = 7'b001_0000;
            default: seg = 7'b111_1111;  // blank
        endcase
    end
endmodule
```

The argmax over 10 values fits in a small priority tree (10-input comparator chain, ~10 LEs). It operates on the 10 output values after they land in the UB during the DONE state.

## LED Status Encoding

| LEDR bits | State |
|---|---|
| `10'b00_0000_0001` | IDLE — waiting |
| `10'b00_0000_0010` | LOAD_WEIGHTS |
| `10'b00_0000_0100` | WEIGHT_DRAIN |
| `10'b00_0000_1000` | STREAM_ACT |
| `10'b00_0001_0000` | WAIT_DRAIN |
| `10'b00_0010_0000` | DONE |
| `10'b10_0000_0000` | ERROR (any pipeline fault) |

## Quartus Project Setup

1. **New project:** Device = `5CSEMA5F31C6`, top-level entity = `tpu_top`
2. **Source files:** Add all `rtl/*.sv` to the project
3. **Pin assignments:** Import `DE1-SoC.qsf` or run `pin_assignment_DE1_SoC.tcl` in the Tcl console
4. **Platform Designer:** Add HPS component → enable LWH2F bridge → connect to `host_interface` → Generate HDL → add `.qip` to project
5. **Compile:** Analysis + Synthesis + Fitter + Timing Analyzer
6. **Timing target:** 50 MHz (may PLL to 100 MHz for the datapath; keep LWH2F at 50 MHz)
7. **Resource check:** After compile, confirm DSP count ≤ 87, M10K ≤ 553, ALMs ≤ 32,070

## Testbench Plan

`tests/tpu_top_tb.sv` — end-to-end simulation (Icarus, no physical hardware):

1. **Initialize weight ROM** with a known 2×2 weight matrix via `$readmemh`.
2. **Simulate host_interface** by driving AXI4-Lite transactions directly (no ARM core in simulation).
3. **Write input activation matrix** via simulated host write transactions.
4. **Assert START**, wait for DONE.
5. **Read output** via simulated host read transactions.
6. **Compare** against a Python/NumPy reference for the same int8 inputs.

This testbench validates the entire end-to-end path from weight load to ReLU output, including the FSM sequencing, without requiring hardware.
